Installing Confluent

2021-12-05
2 minute read

All installations below are for Ubuntu

Install CP via Tarball for Local Development

Installation and configuration

 1## Install dependencies
 2sudo apt-get install openjdk-11-jre-headless
 3
 4## Download and extract package
 5curl -O http://packages.confluent.io/archive/6.0/confluent-6.0.0.tar.gz
 6tar xzf confluent-6.0.0.tar.gz
 7
 8## Configure CLI
 9export CONFLUENT_HOME=${HOME}/confluent-6.0.0 \
10    && echo "export CONFLUENT_HOME=$CONFLUENT_HOME" >> ~/.bashrc
11echo "export PATH=$CONFLUENT_HOME/bin:${PATH}" >> ~/.bashrc
12~/confluent-6.0.0/bin/confluent completion bash | sudo tee /etc/bash_completion.d/confluent \
13    && echo "source /etc/bash_completion.d/confluent" >> ~/.bashrc \
14    && source ~/.bashrc

Starting services

1## Start services
2confluent local services start

Check https://docs.confluent.io/confluent-cli/current/command-reference/local/index.html

Produce and Consume Avro data

 1## create Avro schema
 2cat <<EOF > ~/temperature_reading.avsc
 3{
 4 "namespace": "io.confluent.examples",
 5 "type": "record",
 6 "name": "temperature_reading",
 7 "fields": [
 8    {"name": "city", "type": "string"},
 9    {"name": "temp", "type": "int", "doc": "temperature in Fahrenheit"} ]
10}
11EOF
12
13## Start a command line producer to produce events
14confluent local services \
15    kafka produce temperatures \
16    --property parse.key=true --property key.separator=, \
17    --property key.serializer=org.apache.kafka.common.serialization.StringSerializer \
18    --value-format avro \
19    --property value.schema.file=$HOME/temperature_reading.avsc
20
21
22## In a new window, setup command line consumer
23confluent local services \
24    kafka consume temperatures \
25    --property print.key=true \
26    --property key.deserializer=org.apache.kafka.common.serialization.StringDeserializer \
27    --value-format avro

Sample correct input

1alameda,{"city":"alameda","temp":58}
2ashland,{"city":"ashland","temp":62}
3nairobi,{"city":"nairobi","temp":65}
4sydney,{"city":"sydney","temp":75}

Sample incorrect value

1jaipur,{"city":"jaipur","temp":"87"}

Destroy local CP

1confluent local destroy

Explore run-class Scripts and Environment Variables

  1. systemd invokes the kafka-server-start script, which takes a configuration file /etc/kafka/server.properties as an argument.

  2. The kafka-server-start invokes the kafka-run-class script, which does most of the “heavy lifting”.

  3. In the kafka-run-class script, environment variables are used to customize the JVM options passed to the java command.

  4. The CLASSPATH variable is used to load classes from .jar files located under various subdirectories of /usr/share/java/

  5. The java process starts and server logs are output to various .log files under /var/log/kafka/.

Property files

  • ZooKeeper: /etc/kafka/zookeeper.properties
  • Kafka Broker: /etc/kafka/server.properties
  • Schema Registry: /etc/schema-registry/schema-registry.properties
  • ksqlDB: /etc/ksqldb/ksql-server.properties
  • Kafka Connect: /etc/schema-registry/connect-avro-distributed.properties
  • Confluent REST Proxy: /etc/kafka-rest/kafka-rest.properties
